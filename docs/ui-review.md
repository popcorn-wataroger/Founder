# UI確認方法

画面やスタイルを変更したときは、PR を出す前に必ず自分で UI を確認します。確認した内容は PR の「動作確認」に書いてください。

## 起動方法

依存関係を同期してから、開発サーバーを起動します。

```bash
uv sync
uv run uvicorn app.main:app --reload
```

ブラウザで次のURLを開きます。

```text
http://localhost:8000
```

## ログイン確認

このプロジェクトは、社員向け画面と管理者向け画面の両方を確認します。

| コード | 確認する画面 |
|--------|--------------|
| `ADMIN` | 管理者画面 |
| `EMP001` | 社員チャット画面 |

パスワードは任意です。

## 基本チェック

最低限、以下を確認します。

- 画面が真っ白になっていない
- ログインできる
- ボタンやリンクを押してもエラーにならない
- 表示崩れや文字の重なりがない
- 入力欄に文字を入力できる
- モーダルを開閉できる
- スクロールできる部分が正しく動く
- ブラウザの開発者ツール Console に明らかなエラーが出ていない
- ターミナルに Python のエラーが出ていない

## 社員画面の確認

`EMP001` でログインして確認します。

- チャット画面が表示される
- メッセージ入力欄に入力できる
- 送信ボタンでメッセージを送れる
- 自分のメッセージとAIの返答が区別して表示される
- 長い文章でも吹き出しや画面が崩れない
- サイドメニューや戻る操作がある場合、期待通りに動く

## 管理者画面の確認

`ADMIN` でログインして確認します。

- 管理者ホームが表示される
- ソース管理画面へ移動できる
- スタッフ一覧画面へ移動できる
- 社員データ画面へ移動できる
- 「このスタッフについてチャット」のモーダルを開閉できる
- トーク全文のモーダルを開閉できる
- カード、テーブル、ボタン、入力欄の表示が崩れていない

## 画面サイズの確認

可能なら、ブラウザ幅を変えて確認します。

| サイズ | 目安 |
|--------|------|
| PC | 幅 1200px 前後 |
| タブレット | 幅 768px 前後 |
| スマホ | 幅 390px 前後 |

確認すること:

- 横スクロールが不要な画面で発生していない
- ボタンの文字がはみ出していない
- 重要な操作ボタンが画面外に隠れていない
- モーダルが画面内に収まっている
- スマホ幅でもタップしやすい余白がある

## PRに書く例

PR の「動作確認」には、実際に確認した内容を書きます。

```markdown
## 動作確認
- `uv run uvicorn app.main:app --reload` で起動
- `EMP001` で社員チャット画面を確認
- `ADMIN` で管理者ホーム、ソース管理、スタッフ一覧を確認
- PC幅とスマホ幅で表示崩れがないことを確認
- Console にエラーが出ていないことを確認
```

## スクリーンショット

UIを変更したPRでは、できるだけスクリーンショットを添付します。

添付するとよいもの:

- 変更した画面
- モーダルやメニューを開いた状態
- PC幅の表示
- スマホ幅の表示
- エラー表示を変更した場合は、エラーが出ている状態

スクリーンショットがあると、レビューする人が変更内容を確認しやすくなります。

## 困ったとき

画面が表示されない場合は、以下を確認します。

```bash
uv sync
uv run uvicorn app.main:app --reload
```

それでも解決しない場合は、以下を共有して相談してください。

- 操作した手順
- 表示されたエラーメッセージ
- ブラウザ Console のエラー
- ターミナルのエラー
- 画面のスクリーンショット
